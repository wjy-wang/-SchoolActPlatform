# 校园活动发布平台用户故事和验收标准

## 1. 用户管理模块

### 1.1 用户注册

- **用户故事**：作为学生，我希望能通过学号和手机号注册账号，从而成为平台用户并参与活动
- **优先级**：Must
- **验收标准**：
  - Given: 学生访问注册页面
  - When: 输入有效的学号、手机号、密码并提交
  - Then: 注册成功并自动登录系统

### 1.2 用户登录

- **用户故事**：作为平台用户，我希望能通过账号密码登录系统，从而访问我的个人信息和活动
- **优先级**：Must
- **验收标准**：
  - Given: 用户访问登录页面
  - When: 输入正确的账号和密码并提交
  - Then: 登录成功并跳转到首页

### 1.3 个人信息管理

- **用户故事**：作为平台用户，我希望能修改个人资料和密码，查看我的报名记录和收藏记录，从而管理我的个人信息
- **优先级**：Must
- **验收标准**：
  - Given: 用户登录系统并进入个人中心
  - When: 修改个人资料并保存
  - Then: 个人资料更新成功
  - Given: 用户登录系统并进入个人中心
  - When: 点击查看报名记录
  - Then: 显示用户的所有报名活动列表
  - Given: 用户登录系统并进入个人中心
  - When: 点击查看收藏记录
  - Then: 显示用户的所有收藏活动列表

### 1.4 权限管理

- **用户故事**：作为系统管理员，我希望能根据用户角色分配不同的权限，从而确保系统安全和功能隔离
- **优先级**：Must
- **验收标准**：
  - Given: 系统有不同角色的用户
  - When: 普通用户尝试访问管理员功能
  - Then: 系统拒绝访问并提示权限不足
  - Given: 系统有不同角色的用户
  - When: 管理员登录系统
  - Then: 管理员可以访问所有功能

## 2. 活动管理模块

### 2.1 发布活动

- **用户故事**：作为管理员，我希望能发布新活动，包括标题、时间、类型、地点、描述、图片等信息，从而向用户展示活动详情
- **优先级**：Must
- **验收标准**：
  - Given: 管理员登录系统并进入活动管理页面
  - When: 填写活动信息并提交
  - Then: 活动发布成功并显示在活动列表中

### 2.2 编辑活动

- **用户故事**：作为管理员，我希望能编辑已发布活动的信息，从而更新活动详情
- **优先级**：Must
- **验收标准**：
  - Given: 管理员登录系统并进入活动管理页面
  - When: 选择已发布的活动并修改信息
  - Then: 活动信息更新成功

### 2.3 删除活动

- **用户故事**：作为管理员，我希望能删除已发布的活动，从而移除不需要的活动信息
- **优先级**：Must
- **验收标准**：
  - Given: 管理员登录系统并进入活动管理页面
  - When: 选择已发布的活动并确认删除
  - Then: 活动删除成功并从列表中消失

### 2.4 活动列表

- **用户故事**：作为平台用户，我希望能浏览所有活动，并支持按时间、类型筛选，从而快速找到感兴趣的活动
- **优先级**：Must
- **验收标准**：
  - Given: 用户访问活动列表页面
  - When: 查看活动列表
  - Then: 显示所有活动信息
  - Given: 用户访问活动列表页面
  - When: 使用筛选功能
  - Then: 显示符合条件的活动列表

### 2.5 活动详情

- **用户故事**：作为平台用户，我希望能查看活动的详细信息，包括报名状态，从而了解活动详情并决定是否报名
- **优先级**：Must
- **验收标准**：
  - Given: 用户访问活动详情页面
  - When: 查看活动信息
  - Then: 显示活动的完整详细信息
  - Given: 用户已登录并查看活动详情
  - When: 查看报名状态
  - Then: 显示用户的报名状态

## 6. 非功能需求

### 6.1 性能需求

- **用户故事**：作为平台用户，我希望系统响应速度快，从而获得良好的使用体验
- **优先级**：Must
- **验收标准**：
  - Given: 用户访问系统
  - When: 加载页面或执行操作
  - Then: 页面加载时间≤2秒，操作响应时间≤1秒
  - Given: 系统有1000人同时在线，50人同时操作
  - When: 执行各种操作
  - Then: 系统运行无明显延迟

### 6.2 安全需求

- **用户故事**：作为平台用户，我希望我的个人信息和操作安全有保障，从而放心使用系统
- **优先级**：Must
- **验收标准**：
  - Given: 用户注册或修改密码
  - When: 系统存储密码
  - Then: 密码以加密形式存储
  - Given: 普通用户尝试访问管理员功能
  - When: 系统验证权限
  - Then: 拒绝访问并提示权限不足
  - Given: 用户输入数据
  - When: 系统处理输入
  - Then: 系统能防止SQL注入、XSS等常见攻击

### 6.3 可用性需求

- **用户故事**：作为平台用户，我希望系统稳定可用，从而随时访问系统
- **优先级**：Must
- **验收标准**：
  - Given: 系统运行中
  - When: 监测系统可用性
  - Then: 系统正常运行时间达到99.9%
  - Given: 系统出现故障
  - When: 进行故障恢复
  - Then: 能在30分钟内恢复正常
  - Given: 系统运行中
  - When: 执行数据备份
  - Then: 系统定期自动备份数据

### 6.4 易用性需求

- **用户故事**：作为平台用户，我希望系统界面友好，操作简单，从而轻松使用系统
- **优先级**：Must
- **验收标准**：
  - Given: 用户访问系统
  - When: 查看界面
  - Then: 界面简洁美观，操作流程清晰
  - Given: 用户使用不同设备访问系统
  - When: 查看页面显示
  - Then: 页面自适应不同屏幕尺寸
  - Given: 用户使用系统功能
  - When: 需要操作指引
  - Then: 系统提供清晰的操作指引和帮助信息
